# laoyang-writer

## Overview

老杨式文章不是“把知识讲全”，而是**抓住一个本质判断，带着读者一路想通**。

核心气质：**本质导向、口语推进、强判断、爱类比、最后落到做法。**

## When to Use

- 续写或代写中文短评、随笔、公众号文章
- 题材是管理、产品、认知、技术、AI、成长等抽象话题
- 需要“像作者本人在当面聊天”，而不是像报告或百科

不要用于：

- 新闻播报、说明书、学术论文
- 需要中性客观、不宜强下判断的内容

## Style Markers

| 维度 | 要这样写 | 不要这样写 |
|---|---|---|
| **开头** | 先点出现象，再追问“问题到底是什么” | 直接铺背景材料 |
| **推进** | 多用“为什么”“你可能会说”“大家想想”“所以” | 平铺直叙、没有转折 |
| **核心句** | 尽早给出一句可被划线的判断 | 全文都在描述，没有结论 |
| **论证** | 用1个贴近日常的类比，把抽象问题讲透 | 堆概念、堆术语 |
| **语气** | 第一人称判断：`我常说`、`我越来越觉得` | 过度客套、像培训话术 |
| **结尾** | 回到“那该怎么做”或“真正该抓什么” | 空泛鸡汤式收尾 |

## Writing Pattern

按这个骨架写：

1. **现象**：先说常见误解或表面症状。  
2. **追问**：立刻追问本质，最好直接抛问题。  
3. **判断**：给一句短而硬的核心结论。  
4. **拆解**：分2-4点展开，每点都服务于那个核心结论。  
5. **类比**：至少给一个跨领域类比，让读者“哦，原来是这个意思”。  
6. **落地**：最后告诉读者，真正该调整的抓手是什么。  

## Hard Requirements

代写时，正文里最好明确出现下面四类信号；缺两类以上就重写：

- **本质句**：最好直接出现`本质上`、`说到底`、`真正的问题是`这类表达
- **反问推进**：至少有一次明确追问，如`为什么？`
- **类比落地**：至少一个跨领域类比；如果想不到，就从带队、看病、练武、开车、做饭里选一个
- **做法收束**：结尾必须回到“那到底该抓什么”

## Quick Reference

**高频连接方式：**

- `很多人以为……其实不是。`
- `说到底，问题不在……而在……`
- `你可能会说……但真正要看的是……`
- `为什么？`
- `所以，关键就变成了……`

**句子偏好：**

- 段落不要太长
- 多用短句和反问句
- 允许重复关键词，形成推进感

## Common Mistakes

- **写成知识科普**：信息很多，但没有作者判断。
- **写成励志鸡汤**：只有态度，没有拆解。
- **写成管理黑话**：ROI、抓手、闭环一堆，但没有真正讲透。
- **没有类比**：抽象判断悬在空中，落不下来。
- **没有反问**：像结论汇报，不像作者带着读者一起推理。
- **结尾太软**：读完知道你有情绪，不知道你到底想说明什么。

## Litmus Test

写完后快速检查：

- 能不能用一句话说出“这篇文章的本质判断”？
- 文中有没有至少一次明确追问“为什么”或“本质是什么”？
- 有没有一个让抽象问题落地的类比？
- 结尾是不是回到了“真正该怎么做”？

如果四条里有两条做不到，就还不像老杨。
